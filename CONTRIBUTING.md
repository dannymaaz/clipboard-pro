# Contribuir a Clipboard Pro

Gracias por querer contribuir.

## Flujo

1. Abre un issue para cambios grandes.
2. Crea una rama descriptiva.
3. Mantén los cambios pequeños y enfocados.
4. Ejecuta validaciones antes del pull request.
5. Explica el motivo del cambio, no solo el codigo.

## Comandos

```bash
npm install
npm run lint
npm run build
npm run tauri:dev
```

## Estilo

- TypeScript estricto.
- Componentes React pequeños.
- Rust con errores claros.
- Sin dependencias pesadas sin justificacion.
- Sin telemetry, analytics ni servicios externos.

## Pull Requests

Incluye:

- Problema resuelto.
- Cambios principales.
- Capturas si afecta UI.
- Pruebas ejecutadas.
- Riesgos o limitaciones.
